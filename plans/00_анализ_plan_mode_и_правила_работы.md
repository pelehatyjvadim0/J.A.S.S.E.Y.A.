# Анализ plan mode и правила работы

## Что я учел из вашего системного промпта

- **Пошаговая декомпозиция обязательна**: каждый этап нужно разбивать на маленькие, понятные действия.
- **Всегда объяснять "зачем"**: не просто "сделай шаг", а "почему именно этот шаг ведет к цели".
- **Junior-friendly стиль**: формулировки простым языком, без лишней теории.
- **No Code в plan mode**: планы описывают логику и порядок действий, а не готовые куски кода.
- **Минималистичная архитектура**: избегаем сложных абстракций, фокус на результате при минимуме лишнего.
- **Performance First и Zero-Waste**: сначала шаги с наибольшим практическим эффектом и наименьшей операционной сложностью.
- **Проверка edge-cases**: в каждом этапе есть пункт "что может пойти не так" и как заранее это предотвратить.

## Как пользоваться этой папкой планов

- Выполняйте файлы по порядку: `01` -> `08`.
- Внутри каждого файла идите по секциям:
  - цель этапа;
  - результат на выходе;
  - пошаговый сценарий;
  - критерии готовности;
  - риски и как их закрыть.
- Не перепрыгивайте этапы: каждый следующий план опирается на артефакты предыдущего.

## Базовый ритм работы (ежедневный)

1. Открыть текущий этап.
2. Пройти шаги "Подготовка" и "Проверка контекста".
3. Сделать 1-3 задачи реализации.
4. Обновить короткий итог: что сделано, что осталось, первый шаг на завтра.

## Что считается успешным по всей программе

- С утра за 5 минут поднимается рабочее окружение и открывается весь контекст.
- За 1-2 минуты понятно, где остановились и что делать дальше.
- Большая часть рутины старта/завершения дня автоматизирована.
- Агент объясняет свои решения и не теряет рабочую нить между сессиями.
