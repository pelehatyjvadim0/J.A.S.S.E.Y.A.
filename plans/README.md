# Пакет пошаговых планов по внедрению Jarvis-флоу

## Как читать

- Идите строго по порядку файлов.
- На каждом этапе сначала проходите цель и критерии готовности, затем выполняйте шаги.
- Если критерии этапа не выполнены, к следующему этапу не переходите.

## Список планов

1. `00_анализ_plan_mode_и_правила_работы.md` — разбор вашего системного промпта и единый стандарт работы с планами.
2. `01_этап_ядро_оркестрации_start_day_end_day.md` — фундамент: сценарии запуска и завершения дня.
3. `02_этап_mcp_первая_волна_инструментов.md` — подключение базового MCP-стека.
4. `03_этап_оперативная_память_рабочего_дня.md` — память рабочего процесса (вчера/сегодня/блокеры).
5. `04_этап_инфраструктурная_устойчивость.md` — надежность, healthchecks, fallback.
6. `05_этап_качество_памяти_и_retrieval.md` — точность контекста и контроль качества выбора фактов.
7. `06_этап_наблюдаемость_и_контроль_решений.md` — метрики, отчеты, алерты, управляемость.
8. `07_этап_голосовой_контур_и_intent_router.md` — голосовые команды и безопасная маршрутизация.
9. `08_этап_финальная_сборка_jarvis_flow.md` — интеграция всего контура в стабильный ежедневный процесс.

## Быстрый старт на утро

- Откройте `01_этап_ядро_оркестрации_start_day_end_day.md`.
- Реализуйте шаги до выполнения критериев этапа.
- После этого переходите к `02` и дальше по цепочке.
