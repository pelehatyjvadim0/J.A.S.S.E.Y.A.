# Этап 07. Голосовой контур и intent-router

## Цель этапа

Добавить голосовое управление так, чтобы оно ускоряло работу, а не делало флоу хрупким.

## Почему это поздний этап

- Голос полезен только когда ядро и инструменты уже стабильны.
- Иначе вы будете тратить время на отладку UI-слоя вместо результата.

## Результат этапа

- Работают базовые голосовые команды для ежедневных операций.
- Есть безопасная маршрутизация интентов с подтверждением рискованных действий.

## Пошаговый план реализации

### Шаг 1. Определить минимальный набор голосовых интентов
- Рекомендуемый набор:
  - "запусти рабочий день";
  - "где остановились";
  - "обнови план";
  - "что делаем сейчас";
  - "заверши день".
- Зачем:
  - ограниченный словарь проще сделать надежным.
- Проверка:
  - каждая команда стабильно распознается.

### Шаг 2. Описать правила подтверждений
- Что делаете:
  - вводите обязательное подтверждение для потенциально опасных операций.
- Зачем:
  - защита от случайных действий из-за ошибок распознавания.
- Проверка:
  - рискованные действия не выполняются без явного подтверждения.

### Шаг 3. Спроектировать intent-router
- Что делаете:
  - описываете слой, который переводит фразы в структурированные команды.
- Зачем:
  - единое место принятия решения "что именно выполнить".
- Проверка:
  - для каждой фразы есть:
    - распознанный интент;
    - аргументы;
    - целевой MCP/сценарий.

### Шаг 4. Добавить fallback на текст
- Что делаете:
  - если голос распознан неуверенно, система предлагает текстовое уточнение.
- Зачем:
  - не терять контроль и не портить UX ошибками распознавания.
- Проверка:
  - неоднозначные команды не запускают неверные действия.

### Шаг 5. Описать правила TTS-ответов
- Что делаете:
  - определяете, в каких случаях нужен голосовой ответ:
    - подтверждение команды;
    - краткий статус;
    - критичный алерт.
- Зачем:
  - голос должен помогать, а не создавать шум.
- Проверка:
  - ответы короткие, информативные и не мешают рабочему ритму.

## Критерии готовности этапа

- Базовые интенты работают устойчиво.
- Есть защита от ложных срабатываний.
- Голос органично встроен в рабочий поток.

## Типовые ошибки и профилактика

- Ошибка: пытаться покрыть "все возможные фразы" сразу.
  - Профилактика: ограниченный набор интентов + итеративное расширение.
- Ошибка: отсутствие confidence-порогов.
  - Профилактика: при низкой уверенности запрашивать подтверждение.
- Ошибка: длинные голосовые ответы.
  - Профилактика: формат "суть + следующее действие".
